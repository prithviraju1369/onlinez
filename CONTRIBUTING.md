# Contributing to Onlinez.ai

Thank you for your interest in contributing to Onlinez.ai! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/onlinez.ai.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes: `npm run build && npm run dev`
6. Commit your changes: `git commit -m 'Add some feature'`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Create a Pull Request

## Development Guidelines

### Code Style

- Use TypeScript for all new files
- Follow the existing code structure and naming conventions
- Use Tailwind CSS for styling (avoid custom CSS when possible)
- Write meaningful commit messages

### Component Guidelines

- Create reusable components in the `components/` directory
- Use TypeScript interfaces for props
- Include JSDoc comments for complex functions
- Ensure components are responsive and accessible

### Testing

- Test your changes locally before submitting
- Ensure the build passes: `npm run build`
- Check for linting errors: `npm run lint`
- Test on multiple screen sizes

### Pull Request Process

1. Update the README.md if needed
2. Ensure your PR description clearly describes the changes
3. Link any related issues
4. Wait for code review and address any feedback

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help others learn and grow

## Questions?

Open an issue for questions or discussions about the project.
